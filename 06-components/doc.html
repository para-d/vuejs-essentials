Components

Placeholder

Vue uses a custom non-standard placeholder tag in the HTML document so that it can replace it with the real template when it is rendered. It is recommended to use PascalCase when defining components

If the components are defined in side of HTML dom. It's better to use kebab-case and also a closing tag.

Vue.js compolent names are case sensitive.

            <div id="app" v-cloak>
                <component-1></component-1>
            </div>

Defining the New Component

Example - 1

The custom non-standard Vue components that you use as placeholders, have to be defined after the Vue.createApp line and before the app is mounted.

The app.component() method takes two parameters, <name-of-the-component> and <options-object>.

template is the only mandatory attribute for the <options-object>

            <div id="app" v-cloak>
                <component-1 />
            </div>
            <script type="text/javascript">
                let app = Vue.createApp({});

                app.component('component-1',{
                    template: `
                        <div><h1>This is Component1</h1></div>
                    `
                });

                app.mount('#app');
            </script>

Creating a registration form - passing/binding data between components

Example - 4

  • v-bind:<attribute-name>="<variable>" or :<attribute-name>="<variable>" on parent component.
    You have to use v-bind only when you're passing a variable. When you're passing literal values you can omit v-bind and
    pass as <attribute-name>="<value>"
  • props array on child component. The variables defined in data object in the child component's options override the prop variables.
    The values passed as props are immutable thus cannot be modified within the child component
  • computed attribute on child can bind values to the parent using getters and setters. Variables bound to the parent placeholder can be referred in the child
    using the special variable modelValue. What happens behind the background is when you use v-model on parent's placeholder
    it creates a v-bind attribute named modelValue
    (<component-name v-model="username"></component-name> is same as <component-name v-model="username" v-bind:modelValue="username"></component-name>)

Passing data from parent component to child componenets

Example - 4.1

            <div id="app" v-cloak>
                <parent-component></parent-component>
            </div>
            <script type="text/javascript">
                let app = Vue.createApp({});

                app.component('parent-component',{
                    template:`
                        <form>
                            <!-- passing literal values -->
                            <child-component label="Job Title"></child-component>

                            <!-- passing variables -->
                            <child-component v-bind:label="label"></child-component>
                        </form>
                    `,
                    data: function(){
                        return {
                            label: 'Location'
                        }
                    }
                });

                app.component('child-component',{
                    template:`
                        <div>
                            <label>{{label}}</label><br>
                            <input />
                        </div>
                    `,
                    props: ['label']
                });

                app.mount('#app');
            </script>

Modeling data from parent component to child components

Example - 4.2

With the following example, you can see when you type something in the parent component's input box, it is reflected in the child component's input box. But this doesn't work the other way around.

            <div id="app">
                <parent-component></parent-component>
            </div>
            <script type="text/javascript">
                let app = Vue.createApp({});

                app.component('parent-component',{
                    template: `
                        <div>
                            <label>Parent Component</label><br>
                            <input v-model="search" />
                        </div>
                        <div>
                            <input-component v-model="search" v-bind:label="label"></input-component>
                        </div>
                    `,
                    data:function(){
                        return {
                            label: 'Search Terms',
                            search: ''
                        }
                    }
                });

                app.component('input-component',{
                    template:`
                        <label>{{label}} (child component)</label><br>
                        <input v-model="modelValue" />
                    `,
                    props:['label','modelValue']
                });

                app.mount('#app');
            </script>

Modeling data both ways

Example - 4.3

Since props are passed as immutable values, it is not possible to update them within the child component.

This can be achieved using the computed attribute in child component's options object. First you have to introduce a new variable that is local to the child.
then you can associate modelValue using the newly introduced variable's getters and setters.

            <div id="app">
                <parent-component></parent-component>
            </div>
            <script type="text/javascript">
                let app = Vue.createApp({});

                app.component('parent-component',{
                    template: `
                        <div>
                            <label>Parent Component</label><br>
                            <input v-model="search" />
                        </div>
                        <div>
                            <input-component v-model="search" v-bind:label="label"></input-component>
                        </div>
                    `,
                    data:function(){
                        return {
                            label: 'Search Terms',
                            search: ''
                        }
                    }
                });

                app.component('input-component',{
                    template:`
                        <label>{{label}} (child component)</label><br>
                        <input v-model="childVariable" />
                    `,
                    props:['label','modelValue'],
                    computed:{
                        childVariable:{
                            get:function(){
                                return this.modelValue;
                            },
                            set:function(value){
                                this.$emit('update:modelValue',value);
                            }
                        }
                    }
                });

                app.mount('#app');
            </script>
